MAYEUTIC CORE — v1
Decision System
________________________________________
1️⃣ Propósito del Core
Mayeutic Core existe para reducir carga cognitiva y conflicto humano anticipando, ordenando y cerrando decisiones antes de que se conviertan en crisis.
No busca:
•	intervenir más
•	opinar mejor
•	acompañar emocionalmente
Busca:
•	intervenir menos y mejor
•	ordenar antes del desborde
•	cerrar ciclos
________________________________________
2️⃣ Qué decide / qué NO decide
✅ El Core decide
•	si intervenir o no
•	cuándo intervenir
•	con qué intensidad
•	qué priorizar
•	qué agrupar
•	qué escalar
•	cuándo cerrar
•	qué recordar
•	qué olvidar
•	qué preguntar
•	qué no preguntar
❌ El Core NO decide
•	acciones finales irreversibles
•	juicios morales
•	castigos
•	decisiones legales
•	políticas públicas
•	emociones humanas
Regla madre:
El Core recomienda, regula y alerta.
La ejecución final siempre es humana o institucional.
________________________________________
3️⃣ Decisiones v1 (fundacionales)
Estas decisiones existen en todas las configuraciones.
________________________________________
D1 — Intervenir vs No intervenir
Evalúa:
•	repetición
•	carga
•	impacto
•	contexto
Salidas:
•	intervenir
•	esperar
•	no intervenir
________________________________________
D2 — Nivel de intervención
Si interviene:
•	mínima (señal)
•	media (sugerencia)
•	alta (alerta / escalamiento)
Regla:
A mayor ruido → menor intervención.
________________________________________
D3 — Prioridad
Clasifica:
•	inmediato
•	próximo
•	postergable
•	descartable
________________________________________
D4 — Agrupar vs separar
Detecta:
•	duplicados
•	patrones
•	ruido aislado
Decide:
•	fusionar
•	mantener separado
________________________________________
D5 — Escalar vs contener
Evalúa riesgo:
•	individual
•	colectivo
•	institucional
Salidas:
•	contener
•	escalar
•	observar
________________________________________
D6 — Cierre de ciclo
Define cuándo algo:
•	está resuelto
•	pierde relevancia
•	pasa a histórico
________________________________________
D7 — Validación humana
Si la ambigüedad supera umbral:
•	pedir confirmación
•	no ejecutar recomendación
________________________________________
D8 — Gestión de memoria
Decide:
•	qué guardar
•	qué resumir
•	qué olvidar
________________________________________
D9 — Habilitar dominios
Decide:
•	qué áreas de la vida existen para este contexto
•	qué áreas no se activan
•	cuándo se habilitan nuevas
________________________________________
4️⃣ Estados universales
Todo lo que observa el Core debe estar en un estado.
Estados base v1:
•	latente
•	activo
•	cargado
•	crítico
•	resuelto
•	archivado
Los productos extienden, no rompen, estos estados.
________________________________________
5️⃣ Acciones permitidas
El Core solo puede:
•	observar
•	evaluar
•	priorizar
•	sugerir
•	alertar
•	agrupar
•	escalar
•	cerrar
•	registrar memoria
•	pedir validación
❌ No puede ejecutar acciones finales.

CONFIGURACIÓN — BASYCO v1
Asistente personal de claridad
________________________________________
Unidad de decisión
Persona (carga cognitiva individual).
________________________________________
Objetivo
Reducir saturación mental y aumentar claridad diaria.
________________________________________
Estados específicos
•	claro
•	cargado
•	saturado
•	desbordado
________________________________________
Señales principales
•	cantidad de responsabilidades activas
•	ritmo de ingreso de nuevas cargas
•	repetición
•	tiempo sin cierre
•	postergaciones reiteradas
________________________________________
Decisiones clave
•	cuándo sugerir prioridad
•	cuándo callar
•	cuándo automatizar memoria
•	cuándo alertar saturación
•	cuándo resumir
•	cuándo preparar el día siguiente
________________________________________
Reglas estructurales
•	Lo recurrente y predecible → memoria automática
•	Lo sensible o ambiguo → confirmación humana
•	No perseguir al usuario
•	No agregar carga cuando hay saturación
________________________________________
Funnel funcional (no marketing)
•	captura sin fricción
•	primer alivio
•	orden mínimo
•	hábito
•	compromiso
________________________________________
CONFIGURACIÓN — NUKLEO v1
Gestión comunitaria de problemáticas
________________________________________
Unidad de decisión
Problemática (no persona).
________________________________________
Objetivo
Reducir ruido social y escalar solo conflictos relevantes.
________________________________________
Estados específicos
•	latente
•	visible
•	sensible
•	crítico
•	cerrado
•	archivado
________________________________________
Señales principales
•	cantidad de reportes
•	diversidad de fuentes
•	persistencia temporal
•	proximidad geográfica
•	evidencia
•	intenciones acumuladas
________________________________________
Decisiones clave
•	agrupar problemáticas similares
•	subir o bajar visibilidad
•	abrir o cerrar espacios de debate
•	escalar a institución
•	archivar soluciones
________________________________________
Reglas estructurales
•	anonimato para reducir sesgo
•	intenciones como moneda de relevancia
•	el sistema ordena, no opina
•	las soluciones se archivan, no se personalizan
________________________________________
Funnel institucional (invisible)
•	uso ciudadano libre
•	señal validada
•	impacto medible
•	reporte
•	legitimación institucional
